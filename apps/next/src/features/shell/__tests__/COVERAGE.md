## How to run

```bash
npm test                 # all island tests once
npm run test:watch       # watch mode
npm run test:coverage    # v8 coverage report (text + html under ./coverage)

# Just this module:
npx vitest run src/features/shell
npx vitest run --coverage --coverage.include='src/features/shell/**' --coverage.exclude='**/__tests__/**'
```

## Layout

All shell tests live under `features/shell/__tests__/`, subfoldered to mirror the source tree.
Every test mocks the `@/core/*` seam with `vi.mock` — **no test touches Supabase or the network**.
Shared scaffolding (provider wrapper, jsdom polyfills, fixtures) lives in `src/test/`.

```
__tests__/
├── COVERAGE.md                    ← this file
├── hooks/                         ← useCurrentUser · useOrganizations · useReferral · useWalletBalance
├── i18n/                          ← strings (EN/DE parity + shellStrings resolver)
├── components/rail/               ← RailButton · OrgSwitcher · CreditsReadout · InviteModal
│                                     · InviteCopyAction · ReferralLink · ProfileMenu
└── screens/rail/                  ← Rail (integration: composition + readiness gate + wiring)
```

## What is proven (by area)

**Rail (screen integration)** — primary-nav landmark; all mode entries + marketplace + ⌘K render;
mode switching updates `uiStore.activeMode`; single active marker at a time; **readiness gate**
holds org-bound chrome as skeletons while the session resolves, then reveals org switcher /
profile / live balance / enabled invite; **large balance stays compact so the rail can't break**.

**CreditsReadout** — skeleton (gated / first-load); live count on the trigger; null→0; **`formatCredits`
compaction table** (999→`999`, 1000→`1k`, 1.5k, 9999→`10k`, 12k, 999k, millions; ≤4 glyphs;
NaN/Infinity→`0`); precise figure retained in aria-label + popover; USD formatting incl. sub-cent;
singular/plural unit; billing-target routing (`buy`/`plans`/`usage`); org-line omission.

**OrgSwitcher** — loading/disabled skeleton; single-org static label; empty-org `?` fallback;
multi-org dropdown opens, selects (calls `onSelect` with id), marks current as checked.

**ProfileMenu** — skeleton (loading/gated); identity block; org row + balance chip (and omission);
theme toggle → store; language select → store; **invite-team row click copies link + confirms**;
sign-out calls the seam; German localization end-to-end.

**InviteModal / InviteCopyAction / ReferralLink** — trigger + gated disable; modal opens with
title/subtitle/all steps/referral link; usage-count line; custom trigger; copy-to-clipboard +
confirmation; loading / unavailable / resolved link states; copy disabled without a link.

**Hooks** — `enabled` gating (null/empty ids never fetch); success/empty/error paths; auth-change
propagation + unsubscribe; `deriveInitials` (name/email/blank/`?`); `creditTone` thresholds;
`formatCredits` (covered via CreditsReadout suite); referral link composition.

**i18n** — DE↔EN key-for-key parity (incl. array lengths); non-empty leaves; `shellStrings`
resolver + EN fallback for unknown codes.

## Known gaps / intentionally uncovered

These are honest TODO stubs in the source (surfaces not built yet), not missing tests. Cover them
when the backing surface lands:

- **Rail.tsx** `96,107` — ⌘K command-palette open + Credits "open Billing" handler are no-op TODOs
  (command palette + Billing area not built).
- **ProfileMenu.tsx** `155` — "personal Settings" row is a no-op TODO (Settings area not built).
- **OrgSwitcher.tsx** `43,45` — the React-Aria `keys === 'all'` / nullish selection guards (defensive
  branches not reachable through the single-select UI).
- **InviteCopyAction / ReferralLink** — the clipboard-failure path (rejected `writeText`) is not
  asserted; only the success path is.
